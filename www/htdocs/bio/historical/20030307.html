This document contains embedded XML/RDF statements defining and describing a vocabulary for representing biographical information.

Introduction

This document describes a vocabulary for describing biographical information about people, both living and dead. It has been designed to be compatible with both RDF and non-RDF XML formats such as FOAF, RSS 1.0 and RSS 2.0. It contains terms useful for finding out more about people and their backgrounds and has some cross-over into genealogical information.

Changes From Previous Version

None, this is the first version.

Namespace

The namespace for the BIO vocabulary is http://purl.org/vocab/bio/0.1/

Vocabulary Classes

Event

This class represents a general event, i.e. something that the person participated in.

Birth

This class represents a birth event.

Marriage

This class represents a marriage event.

Death

This class represents a death event.

Vocabulary Properties

olb

This property contains a one-line biography of the person.

date

This property represents the date of an Event. The date should be formatted as specified in ISO8601. For example: 2003-03-15 corresponds to the 15th March 2003, and 2003-03-15T13:21-05:00 corresponds to 15th March 2003, 8:21 am, US Eastern Standard Time.

place

This property represents the place at which an Event happened. This can be a textual description of the place.

event

This property represents an event associated with the person. This should contain an element that is a subclass of Event.

Examples

Using With FOAF

The BIO vocabulary simply slots into standard FOAF documents:

  
<rdf:RDF
      xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
      xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"
      xmlns:foaf="http://xmlns.com/foaf/0.1/"
      xmlns:bio="http://purl.org/vocab/bio/0.1/"
      >

  <foaf:Person rdf:ID="ian">
    <foaf:name>Ian Davis</foaf:name>
    <foaf:title>Mr</foaf:title>
    <foaf:firstName>Ian</foaf:firstName>
    <foaf:surname>Davis</foaf:surname>
    
    <bio:olb>
        British; software developer; former CTO of Calaba; 
        author of OCS; co-author of RSS 1.0; developer of myRSS and Pepys.
    </bio.olb>
  
    <bio:event>
      <bio:Birth>
        <bio:date>1970-06-15</bio:date>
        <bio:place>Brentwood, Essex, United Kingdom</bio:place>
      </bio:Birth>
    <bio:event>
  
    <bio:event>
      <bio:Marriage>
        <bio:date>1995-08-04</bio:date>
        <bio:place>
          Northampton Register Office, 
          Northampton, Northamptonshire, United Kingdom
        </bio:place>
      </bio:Marriage>
    <bio:event>
  </foaf:Person>
</rdf:RDF>

Using With RSS 1.0

There are a couple of ways to use the BIO vocabulary with RSS 1.0. The first is simply to include the olb property:

<rdf:RDF 
      xmlns="http://purl.org/rss/1.0/"
      xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#" 
      xmlns:dc="http://purl.org/dc/elements/1.1/" 
      xmlns:foaf="http://xmlns.com/foaf/0.1/"
      xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"
      xmlns:bio="http://purl.org/vocab/bio/0.1/"
      >

  <channel rdf:about="http://internetalchemy.org/">
    <title>Internet Alchemy</title>
    <link>http://internetalchemy.org/</link>
    <description>Digital explorations and experiments</description>
    <bio:olb>
        British; software developer; former CTO of Calaba; 
        author of OCS; co-author of RSS 1.0; developer of myRSS and Pepys.
    </bio.olb>
  </channel>    
</rdf:RDF>

The second is to include a snippet of FOAF in a dc:creator property:

<rdf:RDF 
      xmlns="http://purl.org/rss/1.0/"
      xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#" 
      xmlns:dc="http://purl.org/dc/elements/1.1/" 
      xmlns:foaf="http://xmlns.com/foaf/0.1/"
      xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"
      xmlns:bio="http://purl.org/vocab/bio/0.1/"
      >

  <channel rdf:about="http://internetalchemy.org/">
    <title>Internet Alchemy</title>
    <link>http://internetalchemy.org/</link>
    <description>Digital explorations and experiments</description>
    <dc:creator>
      <foaf:Person>
        <foaf:name>Ian Davis</foaf:name>
        <bio:olb>
            British; software developer; former CTO of Calaba; 
            author of OCS; co-author of RSS 1.0; developer of myRSS and Pepys.
        </bio.olb>
        <foaf:mbox_sha1sum rdf:value="69e31bbcf58d432950127593e292a55975bc55fd" />
        <rdfs:seeAlso rdf:resource="http://internetalchemy.org/iand/foaf.rdf" />
      </foaf:Person>
    </dc:creator>
  </channel>    
</rdf:RDF>

Using with RSS 2.0

The BIO vocabulary can be used with RSS 2.0 if care is taken to avoid any specific RDF extensions. This essentially means that the date and place properties must contain only plain text. The most common use, however, will be the addition of the olb tag:


<rss version="2.0" xmlns:bio="http://purl.org/vocab/bio/0.1/">
	<channel>
    <title>Ian's Channel</title>
    <link>http://internetalchemy.org/</link>
    <description>Ian's weblog</description>
    <bio:olb>
      British; software developer; former CTO of Calaba; 
      author of OCS; co-author of RSS 1.0; developer of myRSS and Pepys.
    </bio:olb>
    <item>
      <title>Lack of Imagination</title>
      <description>Here's an RSS item</description>
    </item>
  </channel>
</rss>

Using with HTML

The olb property can be embedded within HTML using meta tags:


<html>
  <head>
    <title>Ian's home page</title>
    <meta name="bio.olb" content="British; software developer; former CTO of Calaba; author of OCS; co-author of RSS 1.0; developer of myRSS and Pepys." />
  </head>
  <body>
    <h1>Ian's home page</h1>
  </body>
</html>

RDF Schema

The RDF schema is also embedded in the source of this document.


<rdf:RDF 
  xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#" 
  xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#" 
  xmlns:owl="http://www.w3.org/2002/07/owl#"
  xmlns:bio="http://purl.org/vocab/bio/0.1/" 
  xmlns:dc="http://purl.org/dc/elements/1.1/">


  <owl:Ontology rdf:about="http://purl.org/vocab/bio/0.1/"
                dc:title="Biographical information vocabulary"
                dc:description="A vocabulary for biographical information, described using W3C RDF Schema and the Web Ontology Language."
                dc:date="2003-03-15" >
    <owl:imports rdf:resource="http://www.w3.org/2000/01/rdf-schema"/>
    <owl:imports rdf:resource="http://www.w3.org/2002/07/owl"/>
    <owl:imports rdf:resource="http://xmlns.com/foaf/0.1/"/>
  </owl:Ontology>
  
  
  <rdfs:Class rdf:about="http://purl.org/vocab/bio/0.1/Event" 
              rdfs:label="Event"
              rdfs:comment="A general event, i.e. something that the person participated in.">

    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>
  </rdfs:Class>


  <rdfs:Class rdf:about="http://purl.org/vocab/bio/0.1/Birth" 
              rdfs:label="Birth"
              rdfs:comment="A birth event.">

    <rdfs:subClassOf rdf:resource="http://purl.org/vocab/bio/0.1/Event"/>
    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>
  </rdfs:Class>

  <rdfs:Class rdf:about="http://purl.org/vocab/bio/0.1/Marriage" 
              rdfs:label="Marriage"
              rdfs:comment="A marriage event.">

    <rdfs:subClassOf rdf:resource="http://purl.org/vocab/bio/0.1/Event"/>
    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>
  </rdfs:Class>

  <rdfs:Class rdf:about="http://purl.org/vocab/bio/0.1/Death" 
              rdfs:label="Death"
              rdfs:comment="A death event.">

    <rdfs:subClassOf rdf:resource="http://purl.org/vocab/bio/0.1/Event"/>
    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>
  </rdfs:Class>


  <rdf:Property rdf:about="http://purl.org/vocab/bio/0.1/olb" 
                rdfs:label="One-line bio"
                rdfs:comment="A one-line biography of the person.">
    
    <rdfs:domain rdf:resource="http://xmlns.com/foaf/0.1/Person"/>
    <rdfs:range rdf:resource="http://www.w3.org/2000/01/rdf-schema#Literal"/>
    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>

  </rdf:Property>

  <rdf:Property rdf:about="http://purl.org/vocab/bio/0.1/date" 
                rdfs:label="Date"
                rdfs:comment="The date of an event.">
    
    <rdfs:domain rdf:resource="http://purl.org/vocab/bio/0.1/Event"/>
    <rdfs:subPropertyOf rdf:resource="http://purl.org/dc/elements/1.1/date"/>
    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>

  </rdf:Property>


  <rdf:Property rdf:about="http://purl.org/vocab/bio/0.1/place" 
                rdfs:label="Place"
                rdfs:comment="The place of an event.">
    
    <rdfs:domain rdf:resource="http://purl.org/vocab/bio/0.1/Event"/>
    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>

  </rdf:Property>

  <rdf:Property rdf:about="http://purl.org/vocab/bio/0.1/event" 
                rdfs:label="Event"
                rdfs:comment="An event associated with a person.">
    
    <rdfs:domain rdf:resource="http://xmlns.com/foaf/0.1/Person"/>
    <rdfs:range rdf:resource="http://purl.org/vocab/bio/0.1/Event"/>
    <rdfs:isDefinedBy rdf:resource="http://purl.org/vocab/bio/0.1/"/>

  </rdf:Property>


</rdf:RDF>
